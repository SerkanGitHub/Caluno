# REFERENCE_MyDuty.md – Competitive Reference for Caluno

This document summarizes MyDuty’s features, strengths, weaknesses, and user pain points.  
Caluno must use this as a reference to avoid repeating MyDuty’s limitations and to exceed its capabilities.

---

## 1. MyDuty – Core Features
- Basic shift calendar for healthcare workers
- Early/Late/Night shift types
- Monthly view
- Team sharing
- Notifications
- Widgets
- Basic notes
- Vacation tracking

---

## 2. MyDuty – Strengths
- Simple and easy to use
- Popular among nurses and hospital staff
- Lightweight UI
- Good for single‑person shift tracking

---

## 3. MyDuty – Limitations (Critical Pain Points)

### Scheduling Limitations
- Only one shift per day
- No overlapping shifts
- No custom shift times
- No partial availability windows
- No multi‑person overlay view

### Sync & Collaboration Issues
- Sync problems between devices
- No real‑time updates
- No offline‑first support
- No conflict detection

### UX Problems
- Outdated UI
- Cluttered monthly view
- No drag‑and‑drop
- Slow or broken widgets

### Missing Intelligence
- No automatic suggestions
- No predictive availability
- No shared free‑time detection
- No group matching

### Missing Social Features
- No “who can meet today?” feature
- No time‑window matching
- No activity suggestions
