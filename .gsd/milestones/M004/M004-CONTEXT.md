# M004: Predictive assistance and release hardening

**Goal:** Deliver predictive scheduling assistance and harden the product for launch, building on a trustworthy cross-platform core.

## Must-Haves

- Add predictive or anticipatory scheduling features (e.g., smarter ranking, suggestions, or proposals) based on real schedule data and existing authorization boundaries.
- Harden the product for launch: reliability, onboarding, performance, accessibility, observability, and deployment readiness.
- Refine UX for calmness, polish, and fit/finish without introducing noise or gimmicks.
- Maintain all trust, privacy, and authorization constraints established in previous milestones.
- Provide explicit UI and diagnostics for predictive features and hardening outcomes.

## Constraints

- Caluno must not become a social network, chat product, analytics dashboard, enterprise suite, or medical system.
- Predictive features must preserve the calm, focused feel of the core experience.
- All predictive logic must operate on trustworthy, permitted schedule data only.

## Dependencies

- M003 (cross-platform continuity) must be fully complete and validated.

## Proof Level

- This milestone proves: launch readiness and differentiated value
- Real runtime required: yes
- Human/UAT required: yes

## Verification

- All predictive features must be covered by unit and e2e tests.
- Product hardening must be demonstrated via regression, accessibility, and performance tests.
- Launch readiness must be validated by user acceptance and deployment checklists.

## Open Questions

- Which predictive features deliver the most value without harming calmness?
- Which hardening aspects are most critical for launch?
- What are the explicit "launchable" criteria for M004?

## Next Steps
